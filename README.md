Курсовой проект защищен на оценку 7.

В базе данных реализовано 3 роли: пользователь, администратор, менеджер.

Основной функционал:
1. Определение ролей (администратор, менеджер, пользователь).
2. Управление пользователями (создание аккаунта, изменение логина, удаление пользователя).
3. Управление фильмами (добавление, удаление, изменение).
4. Управление расписаниями (добавление, удаление, изменение).
5. Управление залами (добавление, удаление, изменение).
6. Просмотр истории заказов (фильтрация по дате, сумме заказа, статусу).
7. Доступ к данным осуществляется через процедуры и функции.
8. Генерации 100000 в таблицу Orders и экспорт/импорт даннных в JSON.

В качестве технологии была выбрана и реализована следующая: мультимедийные типы данных.
